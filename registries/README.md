# MACP Registries

MACP keeps its core protocol intentionally small. Registries exist so the ecosystem can evolve without identifier collisions or undocumented behavior.

## Registry policy

Each registry entry has a status:

- **permanent** — stable and broadly interoperable
- **provisional** — usable but still settling
- **experimental** — non-standard; do not assume interoperability
- **deprecated** — retained for replay and historical compatibility
